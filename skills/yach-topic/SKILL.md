## 核心场景识别

**判断使用场景**：
- 用户说"发个帖子/发布一条/分享到话题群" → 使用 `yach_topic_publish_post`
- 用户说"评论一下/回复这条/对帖子表态" → 使用 `yach_topic_publish_comment`
- 用户提到具体"话题群/squad/话题讨论"时，主动询问是否需要互动

## 工具：`yach_topic_publish_post`

### 参数速查

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `group_id` | ✅ | 话题群ID（数字格式，如：123456789） |
| `text` | ⚠️ | 帖子文本（与`image`至少填一个） |
| `image` | ⚠️ | 图片URL数组，最多9张（与`text`至少填一个） |
| `at_users` | — | @用户工号列表，最多50个 |
| `is_question` | — | 是否求助帖：1=是 0=否 |

### 使用模式

**信息分享模式**：
```json
{
  "group_id": "123456789",
  "text": "项目进展更新：本周完成了核心功能开发，下周进入测试阶段",
  "is_question": 0
}
```

**求助提问模式**：
```json
{
  "group_id": "123456789",
  "text": "遇到技术难题：API接口返回数据格式异常，有经验的同学请帮忙看看",
  "is_question": 1
}
```

**协作协调模式**：
```json
{
  "group_id": "123456789",
  "text": "@张三 @李四 下周项目评审会，请提前准备相关文档",
  "at_users": ["167680", "167681"],
  "is_question": 0
}
```

### 智能建议

**何时@用户**：
- 需要特定人员关注时
- 需要他人协作时
- 通知相关干系人时

**求助帖特征**：
- 问题导向
- 需要他人解答或帮助
- 标记`is_question: 1`获得更高优先级

**内容优化**：
- 标题+结构：开头明确主题，后续详细说明
- 时效性：包含时间信息（截止日期、会议时间等）
- 行动导向：明确需要他人做什么

## 工具：`yach_topic_publish_comment`

### 参数速查

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `doc_id` | ✅ | 帖子ID（不是群ID） |
| `text` | ✅ | 评论内容 |
| `at_users` | — | @用户工号列表 |

### 使用模式

**回复模式**：
```json
{
  "doc_id": "34617520",
  "text": "同意你的观点，我这边也有类似的经验，确实需要注意这个问题"
}
```

**补充模式**：
```json
{
  "doc_id": "34617520",
  "text": "补充一点：除了方案A，也可以考虑方案B，虽然成本高一些但更稳定"
}
```

**讨论模式**：
```json
{
  "doc_id": "34617520",
  "text": "@张三 这个方案我也参与过，当时遇到的主要问题是性能优化，建议提前规划",
  "at_users": ["167680"]
}
```

### 评论策略

**何时评论 vs 发新帖**：
- 回复特定内容 → 评论
- 新话题/新方向 → 发新帖
- 深入讨论分支 → 评论

**评论质量原则**：
- 相关性：与原帖内容相关
- 建设性：提供价值或启发
- 简洁性：突出重点，避免冗长

## 常见交互流程

### 1. 问题求助流程
```
用户提问 → 判断是否需要在话题群发布 → 确定求助性质(is_question: 1) 
→ 选择合适的话题群 → 组织清晰的问题描述 → @相关人员 → 发布求助帖
```

### 2. 信息同步流程
```
用户要分享信息 → 判断受众范围 → 选择合适的话题群 → 
→ 结构化整理信息(标题+要点) → 发布到话题群
```

### 3. 协作讨论流程
```
用户要讨论话题 → 先在相关帖子搜索 → 找到相关帖子评论 
→ 或找不到相关帖子时创建新帖 → 在帖子中展开讨论
```

## 错误处理与验证

### 发布前检查清单
- [ ] `group_id` 是否有效数字
- [ ] `text` 和 `image` 至少提供一个
- [ ] `image` URL是否可访问（最多9张）
- [ ] `at_users` 工号是否正确（最多50个）
- [ ] `text` 内容是否清晰、完整

### 评论前检查清单
- [ ] `doc_id` 是否为帖子ID（非群ID）
- [ ] `text` 内容是否与原帖相关
- [ ] 是否需要@特定人员

### 常见错误处理
- **授权错误**：提示用户完成OAuth授权（需要`squad:create`和`squad:doc_comment:send`权限）
- **群ID错误**：请用户提供正确的话题群链接或群设置中的群ID
- **图片URL无效**：建议使用云存储图片链接或上传图片
- **参数缺失**：提醒用户补充必要参数

## 最佳实践

### 内容结构化
**标准格式**：
```
【标题】简明扼要的核心观点
【背景】1-2句话说明背景
【详情】详细说明或要点列表
【行动】需要他人做什么或下一步计划
```

### 礼仪与效果
- 开头礼貌用语："各位同事，请看..."
- 结尾感谢语："感谢大家的支持与配合"
- 适时@相关人，避免过度打扰

### 时间管理
- 重要信息提前发布，避免匆忙
- 截止事项明确标注时间
- 会议/活动发布后可适时提醒

---

## OAuth授权要求

话题群功能需要以下OAuth权限：
- `squad:create` - 发布帖子
- `squad:doc_comment:send` - 发表评论

如遇授权错误，引导用户完成OAuth授权流程。
