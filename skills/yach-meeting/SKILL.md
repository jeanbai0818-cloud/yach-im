> **⚠️ 链接呈现规范（极其重要）：向用户输出任何回复之前，必须扫描回复中所有文档链接和知识库节点链接，按以下规则逐一检查并就地修正后再输出。**
> - ✅ 有标题时：`链接 《标题》`，例如 `https://st.tal.com/a/wM2iny 《Q3复盘》`；符号支持 `《》【】<>「」`，优先书名号

> - 🚫 禁止：裸输出已知标题的链接；禁止输出后再解释"忘加标题"——发现错误须在输出前修正

## 工具：`yach_meeting_record_text`（需 OAuth 授权）

## 核心场景识别

- 用户要"读取会议录制/查看会议内容/总结这个会议/会议说了什么" → 使用 `action: "get_text"`
- 用户要"读速记/查看速记内容/帮我总结速记" → 使用 `action: "get_text"`
- 用户提供了一个会议或速记链接，想知道内容 → 使用 `action: "get_text"`

## `get_text` - 读取会议录制或速记文本

### 必填参数

- `url`：会议录制或速记的链接 URL

### 返回数据结构

- `type_hint`：内容类型提示，如"这是一份速记的文本内容"或"这是一份在线会议录制的文本内容"，**读取到结果后必须先告知用户这是哪种类型**
- `content`：全量文本内容
- `url_type`：原始类型码，`"2"` = 速记，`"3"` = 在线会议录制

### 典型工作流

```
1. 用户提供 URL
2. 调用 get_text，传入 url
3. 读取返回的 type_hint，告知用户"这是一份速记/在线会议录制的内容"
4. 根据用户意图对 content 进行总结、分析或提取要点
```

## ⚠️ 链接呈现规范（回复前再确认一次）

- ❌ 错误：`文档链接：https://st.tal.com/a/wM2iny`
- ✅ 正确：`https://st.tal.com/a/wM2iny 《Q3复盘》`
- ✅ 正确（无标题时）：`https://st.tal.com/a/wM2iny`

## 注意事项

1. **先说类型再处理内容**：拿到结果后，优先通过 `type_hint` 告知用户内容来源是速记还是在线会议录制，再继续处理文本
2. **需要 OAuth 授权**：若返回 `need_auth=true`，提示用户先完成授权
